ZuzitaErrorUI Demo

Desarrollo móvil sin consola tradicional · Intercepta errores, monitorea red y muestra UI amigable

🔍 4️⃣0️⃣4️⃣ No encontrado

Simula un recurso inexistente, como una API o página perdida. Muestra el ícono 404 y mensaje amigable.

📡 Offline Sin red

Simula la pérdida de conexión a internet. El sistema reacciona automáticamente al evento 'offline'.

🐛 JavaScript Error

Lanza una excepción real en el código o causa un ReferenceError para activar la UI de errores.

🖥️ Error servidor

Simula respuesta 500 desde un fetch o muestra error de conexión al backend.

🚧 Página vacía

Muestra el mensaje de "contenido no encontrado" — útil para SPAs sin root.

📋 Utilidades + Log

Log manual, toggle consola, y ver estado de errores registrados.

✨ ¿Qué hace ZuzitaErrorUI?

  • ✅ Intercepta window.onerror y promesas rechazadas.
  • ✅ Detecta cambios en conexión (offline/online).
  • ✅ Monitorea fetch para códigos 404 y 500.
  • ✅ Consola visual flotante accesible desde móvil.
  • ✅ Mensajes amigables según tipo de error + botón reintentar.
// Uso manual desde consola del navegador
ZuzitaErrorUI.mostrar('server', 'Error simulado desde consola');
ZuzitaErrorUI.log('Mensaje personalizado');
ZuzitaErrorUI.toggleConsole();
